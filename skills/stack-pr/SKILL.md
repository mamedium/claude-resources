# Stacked PR Management

Manage branch stacks for stacked PR workflows. Tracks which branch each feature branch is based on, enabling coordinated rebases across the entire stack.

## Stack File Location

Stack configuration is stored at: `~/.stack/<repo-name>.yaml`

The repo name is derived from the git remote origin URL or the directory name if no remote exists.

## Commands

| Command | Description |
|---------|-------------|
| `/stack-pr` or `/stack-pr status` | Show current stack visualization |
| `/stack-pr add` | Add current branch to stack (prompts for base) |
| `/stack-pr add <base>` | Add current branch stacked on `<base>` |
| `/stack-pr update` | Fetch and rebase current branch onto its base |
| `/stack-pr sync` | Recursively rebase entire stack from root down |
| `/stack-pr delete` | Remove current branch from stack config |

## Stack File Format

```yaml
# ~/.stack/agents.yaml
# Auto-generated by /stack-pr - safe to edit manually

branches:
  feature/auth:
    base: main

  feature/auth-ui:
    base: feature/auth

  feature/auth-api:
    base: feature/auth

  feature/auth-api-tests:
    base: feature/auth-api
```

This represents:
```
main
└── feature/auth
    ├── feature/auth-ui
    └── feature/auth-api
        └── feature/auth-api-tests
```

## Workflow: `/stack-pr status` (default)

Show the current stack state with visual tree.

### Step 1: Identify Repository

```bash
# Get repo name from remote
REPO_NAME=$(basename -s .git "$(git remote get-url origin 2>/dev/null)" || basename "$(pwd)")
echo "Repository: $REPO_NAME"
```

### Step 2: Load Stack File

Stack file location: `~/.stack/<repo-name>.yaml`

```bash
STACK_FILE="$HOME/.stack/${REPO_NAME}.yaml"
```

If the file doesn't exist, report "No stack configured for this repository."

### Step 3: Get Current Branch

```bash
CURRENT_BRANCH=$(git branch --show-current)
```

### Step 4: Build and Display Tree

Parse the YAML and build a tree visualization:

```
Stack for: agents
Current branch: feature/auth-ui (*)

main
└── feature/auth
    ├── feature/auth-ui (*)    ← you are here
    └── feature/auth-api
        └── feature/auth-api-tests

Branches in stack: 4
```

Mark the current branch with `(*)` and an arrow indicator.

### Step 5: Check Sync Status

For each branch in the stack, check if it needs rebasing:

```bash
# Check if branch is behind its base
git fetch origin --quiet
git rev-list --count <branch>..origin/<base>
```

If count > 0, mark as "needs rebase":
```
└── feature/auth-ui (*)         ← you are here
    └── feature/auth-api        ⚠ needs rebase (3 commits behind)
```

## Workflow: `/stack-pr add` or `/stack-pr add <base>`

Add the current branch to the stack configuration.

### Step 1: Get Current State

```bash
CURRENT_BRANCH=$(git branch --show-current)
REPO_NAME=$(basename -s .git "$(git remote get-url origin 2>/dev/null)" || basename "$(pwd)")
STACK_FILE="$HOME/.stack/${REPO_NAME}.yaml"
```

### Step 2: Determine Base Branch

If `$ARGUMENTS` contains a base branch (second argument), use it.

Otherwise, detect or prompt:
1. Check if current branch was recently created from another branch
2. Look at merge-base with common bases (main, dev, master)
3. Ask user: "What branch is `<current>` based on?"

**Validation:**
- Base branch must exist (locally or on remote)
- Base branch cannot be the current branch
- Adding should not create a cycle

### Step 3: Ensure Stack Directory Exists

```bash
mkdir -p ~/.stack
```

### Step 4: Update Stack File

If file doesn't exist, create it:
```yaml
# ~/.stack/<repo>.yaml
# Auto-generated by /stack-pr - safe to edit manually

branches: {}
```

Add or update the branch entry:
```yaml
branches:
  <current-branch>:
    base: <base-branch>
```

**IMPORTANT:** Preserve existing entries. Only add/update the current branch.

### Step 5: Confirm

```
Added to stack:
  feature/auth-ui → feature/auth

Updated stack file: ~/.stack/agents.yaml
```

## Workflow: `/stack-pr update`

Rebase the current branch onto its configured base.

### Step 1: Load Configuration

```bash
CURRENT_BRANCH=$(git branch --show-current)
REPO_NAME=$(basename -s .git "$(git remote get-url origin 2>/dev/null)" || basename "$(pwd)")
STACK_FILE="$HOME/.stack/${REPO_NAME}.yaml"
```

Read the stack file and find the base for current branch.

If current branch is not in the stack:
```
Error: Branch 'feature/foo' is not in the stack.
Run `/stack-pr add <base>` to add it first.
```

### Step 2: Fetch Latest

```bash
git fetch origin --prune
```

### Step 3: Check for Uncommitted Changes

```bash
git status --porcelain
```

If there are uncommitted changes:
```
Error: You have uncommitted changes. Please commit or stash them first.
```

### Step 4: Rebase onto Base

```bash
git rebase origin/<base-branch>
```

**If rebase succeeds:**
```
Successfully rebased feature/auth-ui onto feature/auth

Branch is now up to date with its base.
```

**If rebase has conflicts:**
```
Rebase conflict detected!

Conflicting files:
  - src/auth/login.ts
  - src/auth/session.ts

To resolve:
  1. Fix conflicts in the listed files
  2. Run: git add <resolved-files>
  3. Run: git rebase --continue

To abort: git rebase --abort
```

**STOP here if conflicts occur.** Do not attempt automatic resolution.

### Step 5: Notify About Dependent Branches

After successful rebase, check if any branches depend on the current branch:

```
Note: The following branches are stacked on feature/auth-ui:
  - feature/auth-ui-tests

Consider running `/stack-pr sync` to update the entire stack.
```

## Workflow: `/stack-pr sync`

Recursively rebase the entire stack from root to leaves.

### Step 1: Load Configuration

```bash
CURRENT_BRANCH=$(git branch --show-current)
REPO_NAME=$(basename -s .git "$(git remote get-url origin 2>/dev/null)" || basename "$(pwd)")
STACK_FILE="$HOME/.stack/${REPO_NAME}.yaml"
```

### Step 2: Build Dependency Graph

Parse the stack file and build a directed graph:
- Nodes: all branches in the stack
- Edges: branch → base

Identify root branches (branches whose base is not in the stack, e.g., `main`).

### Step 3: Topological Sort

Order branches so that each branch is rebased after its base:

```
Sync order:
  1. feature/auth (base: main)
  2. feature/auth-ui (base: feature/auth)
  3. feature/auth-api (base: feature/auth)
  4. feature/auth-api-tests (base: feature/auth-api)
```

### Step 4: Fetch All

```bash
git fetch origin --prune
```

### Step 5: Check for Uncommitted Changes

```bash
git status --porcelain
```

Abort if uncommitted changes exist.

### Step 6: Rebase Each Branch in Order

For each branch in topological order:

```bash
git checkout <branch>
git rebase origin/<base>  # or local <base> if base is in stack
```

**Progress output:**
```
Syncing stack (4 branches)...

[1/4] feature/auth → main
      ✓ Already up to date

[2/4] feature/auth-ui → feature/auth
      ✓ Rebased (2 commits)

[3/4] feature/auth-api → feature/auth
      ✗ CONFLICT - stopping sync

Conflict in feature/auth-api!
Resolve conflicts, then run `/stack-pr sync` again to continue.
```

**STOP immediately on first conflict.** Do not continue to dependent branches.

### Step 7: Return to Original Branch

After sync completes (or stops due to conflict), return to the original branch:

```bash
git checkout <original-branch>
```

### Step 8: Summary

```
Stack sync complete!

  ✓ feature/auth - up to date
  ✓ feature/auth-ui - rebased
  ✓ feature/auth-api - rebased
  ✓ feature/auth-api-tests - rebased

All 4 branches are now in sync.
```

## Workflow: `/stack-pr delete`

Remove the current branch from the stack configuration.

### Step 1: Get Current State

```bash
CURRENT_BRANCH=$(git branch --show-current)
REPO_NAME=$(basename -s .git "$(git remote get-url origin 2>/dev/null)" || basename "$(pwd)")
STACK_FILE="$HOME/.stack/${REPO_NAME}.yaml"
```

### Step 2: Check for Dependent Branches

Before deleting, check if any branches depend on this one:

```
Warning: The following branches depend on feature/auth:
  - feature/auth-ui
  - feature/auth-api

Deleting feature/auth from the stack will orphan these branches.
```

Ask user to confirm: "Delete anyway? The dependent branches will need their base updated."

### Step 3: Remove from Stack File

Remove the branch entry from the YAML file.

If user confirmed deletion of a branch with dependents, update dependent branches to point to the deleted branch's base:

```yaml
# Before
feature/auth:
  base: main
feature/auth-ui:
  base: feature/auth

# After (feature/auth deleted, dependents re-pointed)
feature/auth-ui:
  base: main
```

### Step 4: Confirm

```
Removed from stack: feature/auth

Updated dependents to point to 'main':
  - feature/auth-ui
  - feature/auth-api

Stack file updated: ~/.stack/agents.yaml
```

## Edge Cases

### Branch Not on Remote

If a branch exists locally but not on remote, use local branch for rebase:
```bash
git rebase <base>  # instead of origin/<base>
```

### Base Branch Deleted

If a branch's configured base no longer exists:
```
Error: Base branch 'feature/old' no longer exists.

Options:
  1. Update the base: /stack-pr add <new-base>
  2. Remove from stack: /stack-pr delete
```

### Circular Dependencies

When adding a branch, detect cycles:
```
Error: Cannot add feature/auth with base feature/auth-ui.
This would create a circular dependency:
  feature/auth-ui → feature/auth → feature/auth-ui
```

### Empty Stack File

If stack file exists but has no branches:
```
Stack is empty for: agents

Use `/stack-pr add <base>` to start tracking branches.
```

### Multiple Stacks (Different Roots)

A repo can have multiple independent stacks:

```
main
├── feature/auth
│   └── feature/auth-ui
└── feature/payments
    └── feature/payments-ui

develop
└── hotfix/urgent
```

Display all stacks in status, grouped by root.

## Conflict Resolution Guide

When a rebase conflict occurs:

1. **Identify conflicts:**
   ```bash
   git status
   ```

2. **Resolve each file:**
   - Edit the file to resolve `<<<<<<<` markers
   - Or use: `git checkout --theirs <file>` / `git checkout --ours <file>`

3. **Mark as resolved:**
   ```bash
   git add <resolved-file>
   ```

4. **Continue rebase:**
   ```bash
   git rebase --continue
   ```

5. **Resume stack sync:**
   ```
   /stack-pr sync
   ```

**To abort and restore original state:**
```bash
git rebase --abort
```

## Helper Script

Use the Python helper script for all YAML operations:

```bash
SCRIPT="$HOME/claude-resources/skills/stack-pr/scripts/stack.py"

# Show stack status
python3 "$SCRIPT" status <repo> [current-branch]

# Get base for a branch (exits 1 if not found)
python3 "$SCRIPT" get-base <repo> <branch>

# Add branch to stack
python3 "$SCRIPT" add <repo> <branch> <base>

# Delete branch from stack (auto-reparents dependents)
python3 "$SCRIPT" delete <repo> <branch>

# List direct dependents
python3 "$SCRIPT" dependents <repo> <branch>

# Get branches in topological order (for sync)
# Output: branch:base per line
python3 "$SCRIPT" topo-sort <repo>

# Validate adding would not create cycle
python3 "$SCRIPT" validate <repo> <branch> <base>
```

The script handles:
- YAML parsing and generation
- Tree building and visualization
- Cycle detection
- Topological sorting for sync order
- Automatic reparenting when deleting

## Critical Rules

1. **NEVER force push** - This skill never runs `git push --force`
2. **STOP on conflicts** - Never attempt automatic conflict resolution
3. **Preserve uncommitted work** - Always check for dirty state before rebasing
4. **Validate before writing** - Check for cycles and missing bases before updating config
5. **Return to original branch** - After sync, always return to where the user started
