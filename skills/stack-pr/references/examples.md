# Stack PR Examples

## Example 1: Building a Stack

Starting from scratch, building a feature with multiple dependent PRs:

```
# Start on main, create first feature branch
git checkout -b feature/auth
# ... make changes ...

# Add to stack
/stack-pr add main

# Stack now:
# main
# └── feature/auth

# Create dependent branch for UI
git checkout -b feature/auth-ui
# ... make changes ...

/stack-pr add feature/auth

# Stack now:
# main
# └── feature/auth
#     └── feature/auth-ui

# Create another branch off auth for API
git checkout feature/auth
git checkout -b feature/auth-api
# ... make changes ...

/stack-pr add feature/auth

# Stack now:
# main
# └── feature/auth
#     ├── feature/auth-api
#     └── feature/auth-ui
```

## Example 2: Syncing After Main Updates

When main gets updated and you need to rebase everything:

```
# Check current stack status
/stack-pr status

# Output:
# Stack for: agents
# Current branch: feature/auth-api (*)
#
# main
# └── feature/auth
#     ├── feature/auth-api (*)  ← you are here
#     └── feature/auth-ui
#
# Branches in stack: 3

# Sync entire stack
/stack-pr sync

# Output:
# Syncing stack (3 branches)...
#
# [1/3] feature/auth → main
#       ✓ Rebased (5 commits from main)
#
# [2/3] feature/auth-api → feature/auth
#       ✓ Already up to date
#
# [3/3] feature/auth-ui → feature/auth
#       ✓ Rebased (2 commits)
#
# Stack sync complete!
```

## Example 3: Handling Conflicts

When a rebase conflict occurs during sync:

```
/stack-pr sync

# Output:
# Syncing stack (3 branches)...
#
# [1/3] feature/auth → main
#       ✓ Rebased
#
# [2/3] feature/auth-api → feature/auth
#       ✗ CONFLICT - stopping sync
#
# Conflict in feature/auth-api!
#
# Conflicting files:
#   - src/auth/api.ts
#
# To resolve:
#   1. Fix conflicts in src/auth/api.ts
#   2. Run: git add src/auth/api.ts
#   3. Run: git rebase --continue
#   4. Run: /stack-pr sync
#
# To abort: git rebase --abort

# After resolving:
git add src/auth/api.ts
git rebase --continue

# Continue sync
/stack-pr sync

# Output:
# Syncing stack (3 branches)...
#
# [1/3] feature/auth → main
#       ✓ Already up to date
#
# [2/3] feature/auth-api → feature/auth
#       ✓ Already up to date
#
# [3/3] feature/auth-ui → feature/auth
#       ✓ Rebased
#
# Stack sync complete!
```

## Example 4: Updating a Single Branch

When you only want to rebase the current branch:

```
git checkout feature/auth-ui

/stack-pr update

# Output:
# Fetching origin...
# Rebasing feature/auth-ui onto feature/auth...
#
# Successfully rebased feature/auth-ui onto feature/auth
#
# Note: The following branches are stacked on feature/auth-ui:
#   (none)
#
# Branch is now up to date with its base.
```

## Example 5: Deleting a Branch from Stack

When a PR is merged and you want to clean up:

```
# feature/auth got merged to main
# Need to update the stack

git checkout feature/auth
/stack-pr delete

# Output:
# Warning: The following branches depend on feature/auth:
#   - feature/auth-api
#   - feature/auth-ui
#
# These will be reparented to: main

# Confirm deletion
# (Claude will ask for confirmation)

# After confirmation:
# Removed from stack: feature/auth
# Reparented to 'main': feature/auth-api, feature/auth-ui
#
# New stack:
# main
# ├── feature/auth-api
# └── feature/auth-ui
```

## Example 6: Multiple Independent Stacks

A repo can have multiple stacks with different roots:

```
/stack-pr status

# Output:
# Stack for: monorepo
# Current branch: hotfix/urgent (*)
#
# main
# └── feature/auth
#     ├── feature/auth-api
#     └── feature/auth-ui
#
# develop
# └── hotfix/urgent (*)  ← you are here
#
# Branches in stack: 4
```

## Example 7: Stack File Format

The file `~/.stack/agents.yaml`:

```yaml
# ~/.stack/agents.yaml
# Auto-generated by /stack-pr - safe to edit manually

branches:
  feature/auth:
    base: main

  feature/auth-api:
    base: feature/auth

  feature/auth-ui:
    base: feature/auth

  hotfix/urgent:
    base: develop
```

You can edit this file manually if needed. The skill will preserve your changes.
